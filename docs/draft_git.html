Git

Git er versjonskontrollsystem (et dataprogram), som oppsto av nødvendighet for sikker og effektiv samarbeidsmetode for utviklere av programvare, spesielt relatert til åpen kildekode programvare. Programvaren ble designet og utviklet av Linus Thorvalds (The Linux Foundation, 2015).

Et dokument er en av artefaktene (Det Norske Akademis Ordbok, 2023a), som homo sapiens bruker for å kommunisere. Om det er et manus for en roman, besvarelse av en skoleoppgave, vitenskapelig artikkel skrevet på et spesifikt "naturlig" språk (f. eks. norsk, engelsk, urdu), juridisk kontrakt om et kjøp av eiendom, eller et program skrevet på et programmeringsspråk (f. eks. Pyret, Python, JavaScript), så avgrenser homo sapiens det ofte med et begrep "dokument" ("fil" fra ca. 1950 om informasjon lagret på en datamaskin som én enhet med ett navn). I vår kontekst (IS-114) definerer vi et dokument som elektronisk tekst som kan lagres, redigeres og gjenfinnes (Det Norske Akademis Ordbok, 2023b). et papir eller et sett med papirer med skriftlig eller trykt informasjon, esp. av offisiell type

h2> Prosjektoblig 03

Individuell del

  1. Installere Git på egen datamaskin. Hva er Git og hvorfor trenger man å bruke det? Hvorfor er det ikke anbefalt å installere en GUI (Graphical User Interface) klient applikasjon for Git?
  2. Gjøre seg kjent med grunnleggende bruk av applikasjonen Terminal i macOS eller Linux (Ubuntu 22.04), og GitBash i MS Windows operativsystemet. Hvilke grunnleggende kommandoer trenges for å kunne se filene, som er lagret på egen datamaskin på kommandolinjen? Hvordan finner jeg de samme filene i Terminal/GitBash som jeg ser i applikasjonen med grafisk grensesnitt (explorer?) for fremvisning av mappene hvor filene er lagret (filsystemet)? Hvorfor er det anbefalt å lære å bruke Git-kommandoer fra kommandolinjen?
  3. Installere en "minimalistisk" tekstredigeringsverktøy, som Sublime Text (for macOS, Linux og MS Windows) eller Notepad++ (kun for MS Windows). Hvorfor anbefales det ikke å bruke IDE (Integrated Development Environment) som Visual Studio Code eller IntelliJ hvis man er nybegynner i systemutvikling?
  4. Lage egen konto på github.com og logge inn. Hvorfor trenger man en konto på Github?
  5. Lage en "personal access token" (selektere alt for "repo") for kontoen. Hvorfor trenger man en slik "token"?
  6. Lage en repository (velge et meningsbærende navn for repository, velge en lisens, velge standard README-fil) på Github. Hva er en repository? Hvorfor skal navn være meningsbærende? Hvorfor velge lisens? Hva betyr README-fil og hvorfor er det anbefalt å ha det i hver repository? Hva betyr det at repository-en er på Github? Hva betyr det at repository har en gren (på engelsk branch) som heter main?
  7. Klone (kopiere) repository-en fra Github på sin egen datamaskin. Hvorfor skal man kopiere repository-en fra Github til egen datamaskin? Hvordan velger man et egnet sted å kopiere repository-en i på egen datamaskin? Hvilke filer og eventuelt mapper som blir kopiert fra Github til egen datamaskin med kommandoet for kloning?
  8. Åpne en minimalistisk tekstredigeringsverktøy på egen datamaskin, åpne mappen hvor repository-en fra Github ble klonet, velge filen README.md, skrive inn eget navn i denne filen og lagre endringene på egen datamaskin.
